Slow Installation because of Certificate Checks

Nov 6, 2009 ·4 min read

A few days ago I noticed that an unattended installation of Citrix XenApp 5 was installing very slowly. When I looked at the various jobs and their installation time in the (Altiris) Deployment Server I saw that it was the Citrix Access Management Console that took almost 45 minutes to install:

It was clear that this wasn’t normal since the install job is taking installing OS components like IIS and all subcomponents, activating Application Server and reboot in around 9 minutes. The installation of Citrix XenApp itsself takes only 14 minutes.

I searched with Google and one of the first links was this knowledge base article from Citrix: Slow Access Management Console Installation on XenApp 5.0. The article clearly describes that the delay is occurred by failing checks for Publisher’s and Server Certificate Revocation (because there’s no Internet Connection) and suggests to turn these checks off. Indeed my servers do not have a direct internet connection so the cause and solution were clear.

And actually I had seen similar issues before in other (non Citrix) installations, some examples are Exchange 2007 (here and here) and SQL Server 2008 (the SQL Installer actually checks if there’s an internet connection in the prerequisites check).

The suggested, manual way, of turning of these checks is to clear the following checkboxes in Internet Explorer’s advanced settings Dialog:

But since I had to do this on many servers I decided it would be better to do it with a little VBS script.

I couldn’t find any documentation on where these settings are stored in the registry so I used Process Monitor and RegShot to determine it. I found that the first setting (Check for publisher’s certificate revocation) is stored in “HKCU\Software\Microsoft\Windows\CurrentVersion\WinTrust\Trust Providers\Software Publishing” in a REG_DWORD value State. When turned it off (it’s on by default) it has a value of 146944, when turned on it has a value of 146432. I concluded that the State value is a combined value for several settings and that this particular setting has the value 512. I think it wouldn’t be wise to hard code the values 146944 and 146432 since I have no idea what other settings might be influenced, adding or subtracting this value is also unsafe since we don’t know if the value is on or off with different values. So the safest approach is to test with logical operators (and/or).

This is my code to test if the value is on or off:

vb.net Download
Function GetChkPubRevocation()
	' Create WshShell Object
	Dim WshShell : Set WshShell = WScript.CreateObject("Wscript.Shell")
	' Read Value from Registry
	Dim intState : intState = WshShell.RegRead(cChkPubRevocationKey)
	' Check bitmask
	GetChkPubRevocation = Not (intState And cChkPubRevocation) = cChkPubRevocation
	Set WshShell = Nothing
End Function

And this is the code to set or unset the value:
vb.net Download
Function SetChkPubRevocation(bSet)
	' Create WshShell Object
	Dim WshShell : Set WshShell = WScript.CreateObject("Wscript.Shell")
	' Read Value from Registry
	Dim intState : intState = WshShell.RegRead(cChkPubRevocationKey)
	If bSet Then
		intState = intState And (Not cChkPubRevocation)
	Else
		intState = intState Or cChkPubRevocation
	End If
	SetChkPubRevocation = WshShell.RegWrite(cChkPubRevocationKey, intState, "REG_DWORD")

	Set WshShell = Nothing
End Function


The registry Key for the Check for server certificate revocation is simpler, it’s stored in “HKCU\Software\Microsoft\Windows\CurrentVersion\Internet Settings\CertificateRevocation” as a REG_DWORD value with the values 0 for off and 1 for on. So the Get and Set functions are really simple:

vb.net Download
Function GetChkSrvRevocation()
	' Create WshShell Object
	Dim WshShell : Set WshShell = WScript.CreateObject("Wscript.Shell")
	' Read Value from Registry
	GetChkSrvRevocation = WshShell.RegRead(cChkSrvRevocationKey) = 1
End Function

Function SetChkSrvRevocation(bSet)
	' Create WshShell Object
	Dim WshShell : Set WshShell = WScript.CreateObject("Wscript.Shell")
	' Write Value to registry
	SetChkSrvRevocation = WshShell.RegWrite(cChkSrvRevocationKey, Int(bSet)*-1, "REG_DWORD")
End Function

And here an example of using the functions:
vb.net Download
WScript.Echo("Is ChkSrvRevocationKey on? " & GetChkSrvRevocation)
WScript.Echo("Is ChkPubRevocation on? " & GetChkPubRevocation)

' Toggle it!
SetChkSrvRevocation(Not GetChkSrvRevocation)
SetChkPubRevocation(Not GetChkPubRevocation)
WScript.Echo("Is ChkSrvRevocationKey on? " & GetChkSrvRevocation)
WScript.Echo("Is ChkPubRevocation on? " & GetChkPubRevocation)
